10 exercices de tracking

Mets en pratique tes connaissances de GTM et GA4 sur le mini-site BiblioTech. À chaque exercice : configure ton conteneur GTM, publie une version, puis vérifie le résultat dans GTM Preview et GA4 DebugView.

À lire avant de commencer. Le site contient déjà des dataLayer.push() sur toutes les interactions clés (page_view, view_item, add_to_cart, view_cart, begin_checkout, purchase, click_cta, form_submit). Tu n'as quasiment rien à coder — ton job est de configurer GTM pour exploiter ces events. Seuls les exercices 1 et 8 demandent de toucher au HTML.

Exercice 1

Intégrer un script GTM

Crée ton conteneur GTM sur tagmanager.google.com, puis copie-colle le code d'installation dans chaque page HTML du site :

  • Le premier bloc dans le <head>
  • Le second bloc juste après l'ouverture du <body>

Le bloc <head> à coller :

<!-- Google Tag Manager -->
<script>(function(w,d,s,l,i){w[l]=w[l]||[];w[l].push({'gtm.start':
new Date().getTime(),event:'gtm.js'});var f=d.getElementsByTagName(s)[0],
j=d.createElement(s),dl=l!='dataLayer'?'&l='+l:'';j.async=true;j.src=
'https://www.googletagmanager.com/gtm.js?id='+i+dl;f.parentNode.insertBefore(j,f);
})(window,document,'script','dataLayer','GTM-XXXXXX');</script>
<!-- End Google Tag Manager -->

Tu vas devoir coller ce code dans 6 pages. Frustrant, non ? Garde cette douleur en tête, on verra à la fin du cours comment éviter cette duplication.

Exercice 2

Tracker tous les clics

Configure ton conteneur GTM pour suivre tous les clics sur la page d'accueil. Aucune modification de code, tout se passe dans l'interface GTM.

Indice : déclencheur built-in « Click — All Elements », puis une balise GA4 event avec un nom personnalisé.

Exercice 3

Tracker uniquement le CTA 2

Affine ta balise précédente pour qu'elle ne se déclenche QUE sur le bouton « CTA 2 » de la page d'accueil. Le « CTA 1 » ne doit rien envoyer.

Deux approches possibles :

  • a. Configure une variable GTM basée sur le texte cliqué (Click Text)
  • b. Ou utilise un sélecteur CSS #cta-2 comme condition de déclencheur

Exercice 4

Tracker toutes les pages du site

Mets en place le suivi des pages vues sur les 6 pages : accueil, produit, panier, confirmation, contact et blog.

Pour cet exercice, utilise le déclencheur « All Pages » natif de GTM (basé sur l'event interne gtm.js), pas l'event page_view custom du dataLayer. C'est la méthode la plus simple et la plus rapide pour démarrer.

Indice : balise « GA4 Configuration » ou « Google Tag », déclencheur « All Pages ». Une seule balise suffit pour tout le site.

Exercice 5

Tracker les CTA de la page d'accueil

Envoie un event à GA4 chaque fois qu'un visiteur clique sur l'un des CTA de la home, en transmettant le nom du CTA et sa position.

Le site pousse déjà un event click_cta dans le dataLayer à chaque clic. Inspecte script.js et le code de index.html pour voir les paramètres disponibles (cta_name, cta_label, cta_position).

Indice : déclencheur de type Custom Event avec event name click_cta, variables dataLayer pour récupérer les paramètres, puis une balise GA4 event.

Exercice 6

Tracker la partie e-commerce

Implémente le suivi du tunnel d'achat dans GA4 avec les 5 events e-commerce standards :

  • view_item — page produit
  • add_to_cart — clic sur « Ajouter au panier » ou « + » dans le panier
  • view_cart — page panier
  • begin_checkout — clic sur « Valider la commande »
  • purchase — page de confirmation

Les 5 events sont déjà poussés dans le dataLayer avec la structure GA4 attendue (currency, value, items[], transaction_id pour purchase). Le panier est dynamique : ajoute plusieurs fois le produit pour voir les events add_to_cart et remove_from_cart se déclencher avec les bonnes quantités.

Indice : crée 5 déclencheurs Custom Event (un par event), une variable dataLayer ecommerce, et 5 balises GA4 event qui transmettent l'objet ecommerce complet.

Exercice 7

Tracker la soumission du formulaire

Envoie un event à GA4 chaque fois qu'un visiteur soumet le formulaire de contact. Inclus le sujet sélectionné dans les paramètres.

Le formulaire est codé avec les bons attributs HTML (action, method, pas de novalidate). Tu as donc deux options : utiliser le déclencheur natif « Form Submission » de GTM (qui capte le submit natif), OU utiliser l'event custom form_submit déjà poussé dans le dataLayer. Compare les deux !

Exercice 8

Ajouter un bandeau cookie

Installe un bandeau de consentement qui apparaît au premier passage sur le site. Tant que l'utilisateur n'a pas cliqué sur « Accepter », aucun tag marketing ne doit se déclencher.

Cet exercice demande de modifier le code (HTML + JS pour le bandeau) ET de configurer GTM pour respecter le consentement.

Bonus : utilise Google Consent Mode v2 pour gérer finement les états de consentement (ad_storage, analytics_storage, etc.).

Exercice 9

Installer le pixel Meta

Installe le pixel Meta (Facebook) via GTM et configure les events principaux :

  • PageView sur toutes les pages
  • ViewContent sur la page produit
  • AddToCart au clic d'ajout au panier
  • Purchase sur la page de confirmation, avec value et currency

Tu peux réutiliser les mêmes déclencheurs Custom Event que pour l'exercice 6. Pas besoin de toucher au code, seulement à GTM.

Indice : crée un compte Meta Business Manager test, récupère ton Pixel ID, et utilise les templates communautaires de la galerie GTM pour gagner du temps.

Exercice 10

Tracker le scroll sur la page blog

La page blog.html contient un long article de fond. Configure un déclencheur GTM « Profondeur de défilement » qui envoie un event GA4 à 25 %, 50 %, 75 % et 100 % de scroll.

Indice : déclencheur built-in « Scroll Depth » de GTM, type Vertical, seuils en pourcentages. Ce déclencheur doit être limité à la page blog uniquement (condition sur Page Path).
